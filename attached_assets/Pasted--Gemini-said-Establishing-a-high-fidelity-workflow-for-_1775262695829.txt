Establishing a high-fidelity workflow for the new retarder proofers requires a shift from "Dough-Centric" tracking to "Inventory-Centric" tracking once items are shaped. Below is the technical plan and code implementation to reorganize the Lamination Studio.

1. Architectural Strategy
The Freezer (Aggregated Inventory): Instead of showing individual dough batches, the "Freezer" section will now aggregate counts by pastryType.

The Boxes (Retarder Proofers): We will treat Box 1, Box 2, and Box 3 as distinct locations. Moving items here requires a "Proof Program Confirmation."

Units of Measure: The UI will now support a toggle or dual-input for Pieces vs. Trays (assuming a standard count per tray based on the pastry type).

// 1. Logic to aggregate Freezer Items
const freezerInventory = useMemo(() => {
  const inventory: Record<string, { totalPieces: number, ids: number[] }> = {};
  frozenDoughs.forEach(d => {
    const type = d.pastryType || "Unknown";
    if (!inventory[type]) inventory[type] = { totalPieces: 0, ids: [] };
    inventory[type].totalPieces += (d.proofPieces || 0);
    inventory[type].ids.push(d.id);
  });
  return inventory;
}, [frozenDoughs]);

// 2. Component for the Retarder Boxes
function RetarderBox({ 
  boxNumber, 
  doughs, 
  isGrayedOut 
}: { 
  boxNumber: number, 
  doughs: LaminationDough[], 
  isGrayedOut?: boolean 
}) {
  const isConfirmed = doughs.every(d => d.boxProgramConfirmed);
  
  return (
    <Card className={isGrayedOut ? "opacity-50 grayscale" : ""}>
      <CardHeader className="bg-slate-100 dark:bg-slate-900">
        <CardTitle className="flex justify-between">
          <span>BOX {boxNumber}</span>
          {doughs.length > 0 && (
            <Badge variant={isConfirmed ? "default" : "destructive"}>
              {isConfirmed ? "Program Set" : "Action Required"}
            </Badge>
          )}
        </CardTitle>
      </CardHeader>
      <CardContent>
        {doughs.length === 0 ? (
          <p className="text-muted-foreground text-center py-4">Box Empty</p>
        ) : (
          <div className="space-y-2">
            {/* Aggregated totals per pastry in this box */}
            {Object.entries(aggregatePastries(doughs)).map(([name, qty]) => (
              <div key={name} className="flex justify-between font-mono">
                <span>{name}</span>
                <span>{qty} pcs</span>
              </div>
            ))}
            
            <div className="pt-4 border-t space-y-2">
               <p className="text-xs font-bold">READY AT: 5:30 AM (Tomorrow)</p>
               {!isConfirmed && (
                 <Button className="w-full bg-orange-600" onClick={() => confirmProgram(boxNumber)}>
                   I CONFIRM THE PROOF PROGRAM IS SET
                 </Button>
               )}
               <Button variant="outline" className="w-full">Bake Off</Button>
            </div>
          </div>
        )}
      </CardContent>
    </Card>
  );
}
4. Key Logic Changes
Moving from Freezer → Box
When a user clicks a unique pastry in the Freezer chart:

Popup: "How many Bear Claws are you moving?"

Input: User enters Trays or Pieces. (Logic converts 1 Tray = X Pieces based on pastry configuration).

Action: The system selects the oldest dough IDs of that type from the freezerInventory and updates their status to box1 or box2.

Retarder Proofer Logic (48hr hold)
To support the 48-hour hold mentioned, the "Ready At" timestamp logic will be:

Standard: Today + 1 day at 05:30:00

48-Hour Hold: Today + 2 days at 05:30:00

Simplified Bake Off
Click Bake on Box 1.

The UI switches to "Selection Mode."

Click "Bear Claw" → Popup "How many?" (Input Pieces or Trays).

Submitting this creates a bakeoff_log entry and updates the lamination dough record to baked status, removing it from the box inventory.

5. Grayed Out "Box 3"
To prepare for the future, Box 3 is rendered using the same component but with a disabled state and a "Coming Soon" badge, ensuring the team is visually prepared for the expansion.
