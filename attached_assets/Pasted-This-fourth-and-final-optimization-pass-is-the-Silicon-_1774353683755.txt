### **R Agent Binary Specification: "Project Heartbeat v4.0"**

This specification is designed for the **R Agent** to execute a total architectural transformation of the scheduling and home-screen modules.

#### **1. The Component "State" (Binary Flags)**
To ensure zero lag in the "iPhone" UI, we use a 16-bit integer to track shift status.
* `0x0001`: Active (Clocked in via Square)
* `0x0002`: Late (Shift start passed, no clock-in)
* `0x0004`: Swappable (In the Shift Bank)
* `0x0008`: Audited (Locked for payroll)

#### **2. UI Architecture: The "Glass" Stack**
* **Engine:** `Framer Motion` + `Tailwind CSS`.
* **Visual Logic:**

* **Interaction:** Implement `useSwipeable` hooks on every grid cell. 
    * `deltaX > 50` ⮕ Trigger `releaseShift()`

#### **3. The "Intelligence" Layer (Back-end Logic)**
* **`predictive_labor_model()`**: R Agent must call the Square `orders` API to calculate `Sales_Per_Labor_Hour` (SPLH). If SPLH > $65, R Agent automatically flags the "Grid" with a recommendation for an extra FOH person.
* **`reimbursement_ledger_link()`**:
